## 测试 6：剧本分段打包

- 回答前先完整吞咽，不让复杂生理动作与长句冲突。
- 笑点的语义节点把焦点或切镜交给听者/第三人的反应，而不是始终停在说话者。
- 句尾形成新的碗位置、手部状态、人物距离或情绪状态，不复位。
- 器械落位、吞咽、脚步等声音 cue 与对应动作同位同序，`【声音设计】` 不重新改写声音发生时机。

## 测试 25：镜头运动不能伪装反站桩

- 先补角色目标驱动的距离、道具、视线或任务动作，再决定是否需要运镜。

- 固定镜头用低幅动作、焦点、环境或声音变化推进；环绕运镜必须产生关系、信息或空间变化。
- 只写当前发生的正向动作；次要人物使用较低幅度，所有动作共同指向下一状态。

## 测试 26：默认技术清洁模板

## 测试 27：CUT 一个明确变化与多个辅助变化

输入摘要：巨兽振翼俯冲，张口咬向角色，角色举盾格挡后被撞飞并落地。

期望输出：

- 不把俯冲、张口、格挡、撞飞和落地并列塞进同一个 CUT。
- 每个 CUT 只承担一个能用一句话说清的明确变化。
- 同一 CUT 可以同时包含翼拍、下降运镜、影子扫地、焦点转向巨口、风声增强和尘屑卷起等多个辅助变化；它们共同建立、推进、确认或接力明确变化。
- 只有第二个同等级叙事结果、新动作目标或结果承受者才进入下一 CUT。
- 顺序保持因果：振翼俯冲 -> 巨口逼近 -> 盾口接触 -> 身体被撞离地面 -> 落地后果。

PASS 标准片段：

```text
CUT 2，从上一镜末端向 screen-right 扇落的翼面接入；本镜明确变化是巨兽由半空俯冲转为贴地扑近。低机位横向跟拍同步下降，翼面掠过 foreground，焦点从翼尖转到同方向逼近的巨口，风声增强并卷起地面尘屑；结尾巨口占据 foreground center，角色持盾位于 midground screen-right。
```

## 测试 29：可读性时长门

## 测试 31：节拍估时后拼成15秒内整段

本包已补强以下潜在漏洞：

- 把画面结构、构图、焦点路径和结尾状态并入时间轴。

- 动作必须按路线、接触点、反击、蓄势、后果和声音确认编译。

- 新增中文输出体验闸：最终回答不再使用英文模板标题，测试报告和索引文件不上传到 GPT Knowledge。

- 新增模型预算分配：角色一致性、动作幅度、场景密度只能有一个主消耗，过载时拆段或降复杂度。
- 新增 retake 协议：生成结果先判断 keep / post / edit / re-roll / rewrite，每次尽量只改一个变量。

- 新增武戏三路由：空手格斗、武器格斗、能量招式按需求静默调用，混合战斗只保一个主路由和最多一个次路由。

- 新增武戏动作密度、动作参考视频质量、兵器重量/握法/不变量、能量招式实体因果和单 beat 主特效限制。
