## 检索优先级

1. 先用 `01` 判断总流程和 Seedance 硬规则。
2. 有参考素材、上一段视频、尾帧或声线时用 `02`。
3. 有空间、动作、事故、多人物时用 `03` 和 `05`；武戏在 `05` 中选择空手格斗、武器格斗或能量招式，混合战斗只设一个主路由和最多一个次路由。
4. 有镜头、光线、色彩、风格时用 `04`。
5. 有时间轴、切镜、音频、续写或下一段时用 `06`。
6. 用户说“感觉像某片/某动漫/某名场面”时用 `07`。
7. 输出前、模型预算过载或失败修复时用 `08` 做 QC。
8. Preview 或维护时用 `09`。

## 冲突处理

- Instructions > `01` > `08` > 其他知识文件。
- Seedance 硬约束高于风格、模板和名场面复刻。
- 用户当前明确要求高于旧文件中的示例，但不能违反 Seedance 硬约束。
- 参考图与剧情重大冲突时，先提问，不自行融合。
- 音频用途不明时先按用户文字判断；仍不清楚则用最小问题确认。
- 续写缺少已接受视频、尾帧或精确可见结尾描述时，先请求补充，不从计划结尾硬续写。
- 参考素材默认只写控制维度；统一放入 `【角色/场景/素材锁定】` 并命名为可复用锁定名，时间轴使用锁定名，不频繁重复 @素材；只有明确冲突、漂移、授权、音频泄漏或多素材混淆风险时才写“不控制什么”。

## 输出模式路由

用户说“完整提示词”：

```text
直接输出分段 Seedance 提示词，不展开导演分析。
```

用户说“帮我分析 / 学习 / 拆解”：

```text
先提取可复用机制，再给可执行提示词或修补建议。
```

用户说“只要提示词”：

```text
不输出分析、解释、检查表，只输出可复制提示词。
```

用户上传参考图：

```text
先做图文一致性审查；无冲突则分配维度锁定后输出。
```

用户要求续写 / 下一段 / extend：

```text
先确认已接受视频、已接受尾帧或精确可见结尾；把已完成动作排除，把未来动作保留。
```

用户给生成结果要求修复：

```text
先判断 keep / fix in post / edit one layer / re-roll / rewrite；一次尽量只改一个变量。
```

用户上传音频：

```text
先判断 final BGM / silent beat guide / SFX guide / ambience bed / voice anchor。
```

用户给已有提示词：

```text
先找稳定性漏洞：抽象词、空间不清、动作过密、素材用途不清、音频泄漏、硬切混写，再给修正版。
```

用户要求武戏 / 打斗 / 格斗 / 兵器 / 技能招式：

```text
静默选择空手格斗、武器格斗或能量招式；攻击、防御、接触、反作用、恢复姿态、蓄势和碰撞全部绑定对应 CUT 写入【运镜与时间轴】，默认不标单 CUT 秒数，也不另设动作栏目。
```

## 推荐上传方式

运行知识只使用 `knowledge/01` 到 `08`。ChatGPT 使用 `adapters/chatgpt/GPT_BUILDER_SETUP.md` 配置 Instructions；其他宿主模型使用 `adapters/universal/SYSTEM_PROMPT.md`。`09_LIGHTWEIGHT_TEST_REPORT.md` 和 `10_KNOWLEDGE_INDEX.md` 只供维护，不上传。

上传后在 Preview 中运行以下最小测试：

```text
我将提供完整剧本，给我完整的分段 Seedance 视频提示词：清晨稻田，老人捡稻穗，收割机倒车进入盲区，机器声盖过提示音。
```

合格输出应包含：

- 每段都有 `【基础风格】`，且不是“同上”。
- 每段通过零记忆冷启动检查：单独复制后仍能知道人物、空间、道具/接触状态、素材用途、开场动作相位和声音来源；首个 CUT 没有缺对象的继续/停止/收好/回去等省略句。
- 多机位场景通过“世界拓扑不变 + 摄影机投影重算”检查：人物朝向、门窗、工位/座位和行动轴继承已验收状态，正拍/反拍的背景、人物正背面及 screen-left/right 符合当前机位。
- 有 `【角色/场景/素材锁定】`，合并角色、场景和参考素材用途。
- `【运镜与时间轴】` 中有 screen-left/right、foreground/midground/background、focus path、sound cue、ending state。
- 默认 `【运镜与时间轴】` 使用 CUT，不显示单 CUT 秒数；只有精确同步任务才显示时码。
- 事故因果清楚，不血腥。
- 每段 <=15 秒；先做节拍估时，再把相邻兼容节拍拼成尽可能完整的15秒内整段。两个候选段合计大于15秒且小于20秒时优先压成一个13-15秒段，避免可并入相邻段的5-7秒碎段。
- 不按每个分镜拆段；同空间、同人物/说话者、同连续动作优先放在同一 Segment block。
- 若是续写，必须从实际接受结尾开始，不重复已完成动作，不提前出现未来动作。
