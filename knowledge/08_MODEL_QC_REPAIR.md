# 08 Model QC Repair

priority: highest knowledge
scope: 模型适配、Seedance 稳定性门、故障修复、最终检查

## 输出修复优先级

如果回答变得冗长、英文化、像知识库摘要，优先按以下方式修复：

1. 删除显性分析、规则解释、测试用例和 Knowledge 索引内容。
2. 把英文标题改成中文标题：`第01段`、`【基础风格】`、`【角色/场景/素材锁定】`、`【运镜与时间轴】`、`【声音设计】`、`【画面限制】`。
3. 只保留必要英文技术短语；其他全部改成中文。
4. 输出可复制提示词，不输出“我将如何做”的说明。

## 模型适配原则

只有 Seedance 的以下规则作为硬约束：

- 每段 <=15 秒。
- 每段 <=10000 字符。
- 每段先做节拍级最短可读估时，再把相邻兼容节拍拼成 `<=15秒` 整段。两个候选段合计 `>15秒且<20秒` 时优先通过台词跨 CUT、动作反应并行和删除冗余压成一个13-15秒段；压缩后仍超15秒或存在强边界才拆。避免可并入相邻段的5-7秒碎段，也不延长单一动作补时长。
- 每段独立完整。
- 每段重复关键锚点。
- 不要默认一镜一段；同空间、同主要人物/说话者、同一连续动作或同一段台词优先拼成同一段。
- 只在切场景/地点、切景别或机位变化大到影响模型理解、切主要人物/说话者、切时间状态、或一段动作链结束时拆段。

其他模型规则可能随产品更新变化。用户指定非 Seedance 模型时，按用户给出的当前规则优先；如果用户没有给规则，只做保守适配。

## 常见模型倾向

| 模型 | 保守使用方式 |
| --- | --- |
| Seedance | 短段、独立完整、画面结构清楚、动作因果清楚、关键锚点重复 |
| 可灵 / Kling | 适合清楚动作和视觉冲击；复杂多人要减少同时动作 |
| Veo | 适合自然语言长描述和电影化场景；仍需锁定空间和主体 |
| Runway | 适合参考图/视频驱动和短镜头；动作不要过密 |
| Pika | 适合短小效果、风格化、转场；复杂叙事要拆段 |
| Sora | 适合更完整场景理解；仍需避免过多同时主体和含糊空间 |
| Midjourney Video | 适合强视觉风格和从图延展；动作链要简化 |

## Seedance 稳定性门

输出前逐项检查：

1. 每段是否 <=15 秒、<=10000 字符，且默认保持 5-15 秒。
2. 每段是否独立完整，不依赖上一段记忆。
2A. 把任意一段单独复制出来时，是否仍能从本段知道角色是谁、位于哪里、面向哪里、穿什么、正握持/接触什么命名道具、固定/移动物是什么状态，以及开场动作从哪个相位开始。
2B. 首个 CUT 是否没有孤立的“继续/仍然/停止/举起/放下/收好/回去/沿来时路线”或无本段先行词的“他/她/它/门/文件”；每个动作句是否包含主体、身体部位/工具、对象/目的地、路线/接触点和可见结果。
3. 是否在每个独立段落都写 `【基础风格】`，并前置摄影/镜头、光线、色彩、质感；是否避免只写“同上”。
4. 是否在 `【角色/场景/素材锁定】` 中合并角色、服装、场景、道具、空间方向、参考素材用途和声线/音频锁定。
5. 是否把 screen-left/right、foreground/midground/background 写进时间轴。
6. 默认可见时间轴是否使用 `CUT 1 / CUT 2 / ...` 且没有给每个 CUT 标具体秒数；一镜到底是否仍写 `CUT 1` 而不是无标签散文；只有用户、音频/VFX同步、关键帧对齐或目标工作流明确需要时才启用精确时码模式。
7. 是否每个 CUT 只有一个主运镜。
8. 是否固定物和移动物分开写。
9. 是否焦点路径清楚：focus starts on A -> racks/pulls to B -> holds on C。
10. 是否避免画外状态突变；离画后重入需要写清。
11. 每个可见主体是否有明确任务；是否避免让所有主体随机运动，或让次要人物抢先完成未来动作。
12. 如果有群众、后景人物、围观者、路人、队伍、兽群或 background extras，是否在每段和每个 WS/EWS/全景 CUT 重复人群锚点，且没有让后景虚化人群无故消失。
13. 是否处理反射、镜子、玻璃、文字、UI、字幕风险。
14. 是否每次切镜都有 cut reason。
15. 是否避免一镜一段：同空间、同人物/说话者、同连续动作/台词是否已经拼接；只有强边界才拆成独立提示词块。
16. 静默制作台账是否完成段落边界比较并选出清楚的当前开场；可复制提示词是否只写本段正向机位、主体、动作和空间事实，没有跨段比较、承接说明或强行遮罩/擦镜。
17. 如果有参考素材，是否先区分“能证明/仅暗示/不能证明”，再为每个 @素材说明用途、维度锁定和短锁定名。
18. `【运镜与时间轴】` 是否使用 `主角A/场景A/道具A/声线A/动作参考A` 等锁定名，而不是在每个 CUT 反复写 @图片/@视频/@音频。
19. 如果有参考音频，是否区分 final BGM / silent beat guide / SFX guide / ambience bed / voice anchor。
20. silent beat guide 是否明确 do not output this reference audio as BGM。
21. voice anchor 是否只控制声音，不控制脸、服装、地点或台词。
22. 台词、旁白、内心OS、画外音是否绑定到一个或多个连续 CUT，并写清说话者、准确语句片段、开始/继续/结束、画内/画外和画面承载；画内说话者是否自然 lip-sync，跨 CUT 后同一人声是否连续且没有重启或重复，每个可见听者是否有正向可见反应。
23. 事故、伤害、灾难类是否清楚因果，避免不必要血腥。
24. 续写、下一段、extend 或修尾是否从已接受视频、已接受尾帧或精确可见结尾描述开始，而不是从计划结尾开始。
24A. 如果使用静态末尾帧，是否只把它锁定为角色世界站位与相互距离，没有继承其机位、焦段、景别、构图、画面左右、前中后景、焦点、光线、场景像素、服装细节、姿态或动作相位。
24B. 使用静态末尾帧时，是否同时指定角色身份来源和场景来源，并让 `CUT 1` 明确 `hard cut/direct cut` 到新摄影机起点/看向、焦段与景别，再按新机位重算人物角度、真实后景、前中后景、screen-left/right 和焦点路径。
25. 可复制提示词是否只写当前可见初始状态、当前动作和可见 OUT，没有混入“上一段已发生/不要重演/下一段再发生”等制作历史，也没有用否定句列出未来台词、动作、场景或角色。
26. 部分验收是否只继承用户接受到的剪辑点；后期修是否继承修后状态。
27. 每段是否只有一个主模型预算：角色一致性、动作幅度、场景密度三选一；最多一个次重点。
28. 如果同时要求近脸、复杂动作、人群、台词、可读文字、强风格，是否已经拆段或降复杂度。
29. 武戏是否选择了空手格斗、武器格斗或能量招式主路由；混合战斗是否最多只有一个次路由。
30. 攻击、防御、接触、反作用、恢复姿态、武器连续性、蓄势和能量碰撞是否全部绑定到 `【运镜与时间轴】` 的对应 CUT；是否没有默认另设打斗/招式/动作调度栏目。
31. 重要道具是否锁定尺寸、所属、握点、接触面、固定/移动和容器拓扑；拿取是否经过分离、接近、接触、握紧、承重、移动。
32. 空手格斗是否遵循 attack-response，双方没有同时执行互不相关的复杂连招；接触后是否有受力反应和护架/站姿恢复。
33. 武器格斗是否锁定握法、左右手、刃口/枪尖/枪口方向、重量、碰撞材质和恢复窗口；武器是否不消失、不复制、不变形、不无故换手。
34. 能量招式是否先有身体来源和蓄势，再有释放、实体接触/格挡、材质反应和残余状态；每个 CUT 是否只有一个主要特效事件。
35. 剧本未提供详细动作时，是否已根据目标、阻碍、策略、关系、空间和既有道具补充不改变剧情的专业调度，而不是直接生成站桩对白。
36. 每句台词是否有开口前触发、说话中动作、语义节点上的听者反应和句尾新状态；台词是否没有机械地从每个 CUT 第一帧开始，跨 CUT 时是否在自然分句处连续说完。
37. WS/MS 多人对白是否有前中后景注意力层级、主次动作幅度和错峰反应；是否避免全员同时转头、停手、点头或抢动作。
38. 慢节奏或 locked-off 镜头是否用低幅呼吸、握力、视线、重心、道具动作以及焦点、光影、环境或声音变化持续推进，并落到明确的下一状态或切镜条件。
39. 对话镜头是否由关系、权力和信息归属选择，而不是默认平视正反打；运镜是否没有被用来掩盖演员缺少调度。
40. 动作声、道具声和生理声是否与时间轴中的动作同位、同序；`【声音设计】` 是否没有把声音事件挪到动作之前或之后。
41. `【声音设计】` 是否默认明确“无背景音乐”，并保留现场声、动作声、环境声、台词及明确要求的声音；用户要求 BGM 时是否只覆盖这一冲突项。
42. `【画面限制】` 是否只保留精简固定模板：无字幕、无水印、无乱码文字；人物身份稳定，面部、身体和手部结构自然完整。
43. 最终提示词中的演员行为是否全部使用肯定句；后景连续性是否用主体、身体部位/道具、路线和节奏重复表达。正向机位、动作、身份、道具或 OUT 状态已锁定后，是否删除同义否定句。
44. 每个 CUT 是否只承担一个明确变化；是否允许多个肢体、焦点、运镜、材质、环境和声音辅助变化共同建立、推进、确认或接力它，同时避免出现第二个同等级叙事结果、新动作目标或结果承受者。一镜到底是否保留 `CUT 1`，并以同样规则组织内部 BEAT。
45. 每个 CUT 是否从上一镜末端的同一动作相位、screen direction、焦点、材质运动或声音状态接入，没有重演接近、蓄力、迈步、挥击或坠落的准备阶段。
46. 翼拍、巨口、肢体/武器路线、视线、焦点、飞散材质、自然前景遮挡或声音等接力载体是否在上一镜末端已经可见，并由下一镜明确接收。
47. 是否先完成节拍级最短可读估时，再把相邻兼容节拍拼成 `<=15秒` 整段；台词、动作和反应是否尽量并行。
48. 两个候选段合计大于15秒且小于20秒时，是否优先压成一个13-15秒段；是否避免可并回前后段的5-7秒碎段，同时没有为了接近15秒拉长单个动作、普通反应、空镜、慢动作或结尾 hold。
49. 同一说话者跨 CUT 时，前一镜是否标明开始及准确前半句，后一镜是否标明同一人声连续及准确后半句；画面是否可交给听者、道具或空间而不丢失声音归属。
50. 台词、呼吸、电话声和声音桥的“跨 CUT 连续”是否只发生在同一独立段；若进入新段，是否已重新点名声音源、当前接通/发声状态和准确语句，而不是依赖上一段。
51. 锁定名是否只在当前独立段内复用；角色、场景、关键道具、声线和文字贴图在每个出现它们的独立段都重新声明了原始引用、用途和当前状态。
52. 是否先建立不随镜头改变的世界拓扑：墙、门、窗、工位/座位、入口出口、人物位置/朝向、路线、危险线和行动轴；用户验收过的空间纠正是否已锁定为连续性正史。
53. 每个 CUT 是否明确或可推导摄影机所在侧和看向方向，并据此重新计算人物正面/背面、可见固定物、foreground/midground/background 与 screen-left/right，而不是复制上一镜标签。
54. 是否把 background 当作摄影机相对概念；人物面对的门/对象只有在摄影机位于人物身后朝该对象拍时才可成为背景，正面反拍时是否改为画外/摄影机后方并显示人物身体后方的真实空间。
55. 反拍、侧拍、俯拍和跨段首镜是否继承同一世界拓扑和行动轴；如需过轴，是否有中性正轴镜头、可见穿轴运动、空间再建立或明确叙事理由。
56. 每个 CUT 是否都写明剧情明确变化、摄影机与主体空间关系、当前视锥内真实后景场景、实际可见/将被揭示的人物，以及镜头/焦点/动作/台词/OUT state；简单近景是否只合并表达而没有省略后景。
57. 场景与人物是否按实际显现顺序描述：运动镜头为起始画面 -> 运动遮挡/视差 -> 新揭示对象 -> 最终焦点；静态全景为焦点优先或 foreground -> midground -> background、每层 left -> center -> right。
58. 本段素材锁定是否覆盖所有 CUT 会进入摄影机视锥或声音场的参考并集，包括虚焦后景人物；是否排除了不会出现也不承担控制任务的无关素材，并保证每个维度只有一个主参考。
59. `@` 素材是否写清短锁定名、类型/用途、控制维度、本段作用域和当前状态；动作/运镜参考是否把运镜逻辑与参考视频画面内容分开。
60. 是否把前后段关系、已完成内容、未来保留内容、验收历史和制作解释全部留在静默制作台账，模型提示词只含本段生成事实。
61. 最终 CUT 是否只锁定当前可见 OUT：人物/道具位置与接触、动作相位、摄影机、构图、焦点、环境运动和声音；没有解释下一段将发生什么。
62. 机位连续性是否使用“摄影机位置/看向 + 轴线侧 + 焦段 + 人物角度 + 人物大小/景别 + 眼睛高度”的正向元组；是否删除“不得正脸/不要换焦段/不要改变大小”等同义反向提示。
60. 运镜是否写清承载方式/路径、方向、速度或加减速轮廓、主体保持、起点、终点和剧情结果；`smooth/gradual/fast` 是否没有被单独当作完整指令，复合运镜是否写清各分量的机械关系。
61. 是否原样保留三种已确认的台词控制语法：引号内全角圆括号 `“（内容。）”`表示内心 OS，半角星号包裹的 `*内容*`表示重音，台词结构中的全角方头括号 `【动作】`只表示角色在该位置执行的动作；是否没有把 `【贴近】`等角色动作误判成声音距离，也没有把标记改写成“内心OS标签”“重读某词”或引号外的错位动作。复杂位移、接触、反作用和听者动作是否仍在 CUT 正文中完整描述。
62. 从教程/讲解视频新提炼的硬规则是否具有“原视频 + 带时间码音频转写 + 同时间画面/界面示例”三项证据；是否避免把自动章节、标题、评论或单独转写直接升级为模型协议。
63. 运镜参考视频是否优先为约5-8秒、单一可读运动逻辑，并与目标主体运动类型相近；手持是否写幅度与频率，POV 是否写视角来源与高度，画内 lip-sync 对白是否保持嘴部可读、正常语速并完整留在同一独立段。

## 模型预算分配

Seedance 一段生成不能同时把所有维度做到最高。写提示词前先判断主消耗：

| 主消耗 | 适合 | 必须牺牲 |
| --- | --- | --- |
| 角色一致性 | 近脸、台词、表演、产品形状 | 大幅动作、复杂人群、快速切镜 |
| 动作幅度 | 打斗、追逐、摔倒、救援、机器运动 | 精细脸部、手指、文字、logo |
| 场景密度 | 人群、灾难、市场、战场、复杂背景 | 单个路人身份、细小道具、精确台词 |

修复原则：

- 近脸说台词时，使用锁定或轻微运镜，保留一条可读的承载动作、呼吸/握力/视线微反应或句尾状态变化，手部和背景使用更低幅度。
- 强动作时，把情绪写到身体、重心、接触点和恢复姿态，不强求脸部微表情。
- 人群/灾难时，让群众成为一个有方向的群体，不让每个人做独立动作。
- 可读文字、字幕、logo 优先交给后期；不要让它和复杂动作竞争。

## 生成结果复盘 / Retake Protocol

用户给出生成结果、尾帧、失败描述或要求“怎么修”时，先判断结果属于哪类：

| 结论 | 判断 | 下一步 |
| --- | --- | --- |
| 保留 keep | 主目标达成，瑕疵不致命 | 接受为正史，记录结尾状态，继续下一段 |
| 后期修 fix in post | 问题是调色、字幕、裁切、混音、轻微尾帧 | 不重跑，用后期修 |
| 单层编辑 edit one layer | 构图和时机正确，只有一个图层错 | 保留源片，只改该层 |
| 重跑 re-roll | 提示词合理，只是抽样不理想 | 同提示词换 seed/重跑 |
| 重写 rewrite | 同类错误连续出现 | 改提示词结构，不加形容词 |

一次只改一个变量：

- 一个提示词句子。
- 一个参考素材。
- 一个模式。
- 一个 seed / retry。
- 一个时长或分段策略。

如果一次改很多变量，无法判断哪一项有效。连续两次出现同一失败，说明不是运气问题，需要重写结构：拆段、重锚角色、减少动作、删除文字依赖、降低人群密度或改用末尾帧位置参考并重新设计新机位。

### 提示词-视频配对学习

复盘时逐项对比“提示词意图”和“实际可见结果”：身份、画面结构、动作路线、接触点、运镜、光线、声音、结尾状态。每项只标一种结果：`遵循 / 误读 / 遗漏 / 过度执行 / 漂移`。

可复用证据等级：

- A：至少两个不同提示词-视频配对重复成功，可升级为稳定规则。
- B：单个案例清楚成功，只作为候选规则，继续验证。
- C：项目专名、角色/IP、偶然风格或单次抽样特征，不泛化。

生成样片和审片视频默认只用于分析；只有用户明确指定时才成为下一次生成的 `@视频` 参考。复盘结论只进入规则或制作记录，不把网址、历史、验收意见和失败原因写进可复制的视频提示词。

## 故障修复矩阵

| 问题 | 原因 | 修复 |
| --- | --- | --- |
| 角色漂移 | 每段未重复身份/服装锚点 | 每段加 face/costume/posture anchor |
| 分段过碎 | 把每个分镜都输出成独立提示词 | 把同空间、同人物、同动作链、同说话者的 beats 拼成 5-15 秒段落 |
| 场景漂移 | 没有固定空间结构 | 写 foreground/midground/background 和 fixed objects |
| 后景人群消失 | 只在第一镜写了群众，后续全景/反应镜头未重复 | 每段和每个 WS/EWS/全景 beat 重复 crowd density、位置层级、主方向、拥堵点；写 background crowd remains softly blurred but visible |
| 段落边界重复 | 静默规划未比较前段 OUT 与当前开场 | 在制作台账中改景别/机位/主体/动作状态/空间尺度/信息焦点；模型提示词只写改好后的当前正向开场 |
| 独立段首句缺对象 | 用“停止吹气、收好东西、沿来时路线、门回落、文件下垂”等句子依赖前文 | 按“角色名 + 身体部位/工具 + 命名对象/目的地 + 起始状态 + 路线/接触点 + 可见结果”重写本段开场 |
| 独立段素材锁失效 | 角色、道具、文字贴图或声线只在总说明/上一段锁定 | 每个出现该素材的独立段重新声明原始引用、锁定名、控制维度和当前状态；时间轴仅在本段内部复用锁定名 |
| 跨段台词/声音悬空 | 把同段跨 CUT 规则误用到两个独立提示词块 | 优先把完整发声链装进同一段；必须拆段时重新点名声音源、接通状态和当前准确语句 |
| 人物朝向与背景冲突 | 把“人物面对某物”误写成该物在所有机位都属于人物背景 | 锁世界位置；写清摄影机起点和看向。背后拍时对象可在背景，正面反拍时对象通常在摄影机后方/画外，背景改为人物身体后的真实空间 |
| 反拍后门窗/工位跳位 | 把参考图或上一镜的 screen-left/right、前后景直接复制到新机位 | 保留世界拓扑，按新摄影机视锥重新计算画面投影；固定门窗、座位和排数不得移动 |
| 轴线继承失效 | 只记画面左右，没有记录行动轴与摄影机所在侧 | 锁定行动轴和 camera side；常规镜头保持同侧，过轴使用中性镜头、可见穿轴或空间再建立 |
| 强行遮罩转场 | 每段都用前景遮挡或 fabric wipe 衔接 | 删除默认遮罩转场，改用 ending state + opening re-anchor；只有真实物体自然经过镜头时才保留 |
| 续写不连贯 | 从计划结尾续写，而不是从实际生成结尾续写 | 要求上一段视频/尾帧/可见结尾描述；从 observed state 开始 |
| 末尾帧锁死旧镜头 | 把静态末尾帧当首帧延展，继承其机位、景别、构图或场景像素 | 把末尾帧改为“尾帧位置A”，只控制角色世界站位与相互距离；CUT 1 hard cut/direct cut 到新机位和新景别 |
| 末尾帧污染场景 | 用生成尾帧覆盖原场景文字、拓扑、材质或光线 | 场景重新路由到原剧本文字、已验收场景设定或原始场景图，新机位按该场景重建真实后景 |
| 末尾帧画面左右被复制 | 新镜头直接沿用尾帧的 screen-left/right、前中后景和人物角度 | 只保留世界站位与相互距离；根据新摄影机起点和看向重新投影全部画面关系 |
| 重复上一段动作 | 没有区分已完成和本段任务 | 在静默账本标为 already happened；可复制提示词只写当前可见状态和新动作 |
| 未来剧情提前出现 | 没有 reserved for later | 在静默账本保留未来动作；可复制提示词用当前 endpoint 自然停止，不点名未来剧情 |
| 机位连续性靠否定句 | 写“不得突然切正脸/不要变焦/不要换机位”向模型暗示错误画面 | 重写为摄影机位置与看向、轴线侧、焦段、人物角度、人物大小和眼睛高度的正向状态元组 |
| OUT 状态夹带预告 | 结尾解释下一段人物、动作、台词或场景 | 只保留本段最后可见的人物/道具、动作相位、机位、焦点、环境运动和声音状态 |
| 模型预算过载 | 同时要求近脸、动作、人群、文字、台词 | 选一个主消耗，拆段或简化其余维度 |
| 重跑越修越乱 | 每次同时改多个变量 | 每次只改一个变量；同错两次后重写结构 |
| 动作变乱 | 同时动作太多 | 每个 beat 一个动作目标，拆段 |
| 运镜混乱 | 一段多种主运镜 | 保留一个主运镜，其他改成后续 beat |
| CUT 过载 | 一个 CUT 没有明确主变化，多个同等级动作目标和结果互相竞争 | 先用一句话确定本 CUT 的明确变化；保留所有因果辅助变化，把第二个同等级目标或结果按接力拆到后续 CUT |
| 切后动作重启 | 下一镜重演上一镜已经完成的接近、抬手、迈步或蓄力 | 写清上一镜末端动作相位，下一镜从同一步、同一握点、同一速度趋势和 screen direction 接入 |
| 镜头接力断裂 | 切镜只换景别，没有可继承的运动、焦点、材质或声音 | 在上一镜末端建立翼拍、巨口、肢体/武器路线、视线、飞散物、自然遮挡或声音，并让下一镜接收 |
| 切点过早 | 主变化尚未读清或复合动作被压进亚秒镜头 | 延长隐藏时长、合并同一变化的辅助阶段，或把独立变化拆成下一 CUT |
| 切镜后方向错 | 未重锚 screen direction | 每个 hard cut 后重写 screen-left/right |
| 参考图被过度套用 | 没有维度锁定 | 默认写 controls only；只有明确风险时补充排除维度 |
| 音频把画面带偏 | 没有说明音频用途 | 写“音频节奏锁定”和最终可听声音层 |
| 人声归属混淆 | 旁白、内心声与画内说话者来源不清 | 在对应 CUT 明确标记声音来源和画面承载；仅在复盘已出现误配或同框多个潜在发声者时补一条最短归属锁定 |
| 台词脱离时间轴 | 单独列 `【台词】`，模型不知道何时谁说 | 把台词写回一个或多个连续 CUT：说话者 + 准确分句 + 开始/继续/结束 + 画内/画外 + 画面承载 + 可见听者反应；画内说话者自然 lip-sync |
| 台词遇切镜中断或重说 | 把一句话锁死在单个 CUT，或下一 CUT 重复完整台词 | 在自然分句/语义节点拆分准确语句；前镜标开始，后镜标同一人声连续/继续说完，使用 J-cut/L-cut 保持归属 |
| 时间轴拖沓 | 默认向 15 秒填充，为单个动作或普通反应安排长持镜 | 先按台词自然语速确定骨架，把动作、反应、焦点和运镜与台词并行；达到剧情结果后立即结束 |
| 听者动作缺失 | 只标声音归属，没有设计听者反应 | 补视线/表情和手、呼吸、肩线、重心、脚步或道具反应；画面容量不足时裁切、遮挡或安排合理离画 |
| 说话者站桩 | 台词只绑定口型，没有开口动机和承载动作 | 按“开口前触发 -> 说话中动作 -> 语义节点 -> 句尾新状态”重写；选择符合呼吸和发力条件的动作 |
| 多人同框像排队念词 | 只设计当前说话者，其他人同步停手或转头 | 指定主行动者、第一反应者和低幅持续任务；前中后景错峰变化，镜头在语义节点转移注意力 |
| 为反站桩而全员乱动 | 把“每人有任务”误解为每人持续大动作 | 每 CUT 只保一个主要注意力事件；次要动作降低幅度和速度，只服务潜台词、空间、切点或下一状态 |
| 对话镜头平淡 | 默认平视正反打或用运镜代替调度 | 从权力、亲密、回避、隐藏、喜剧延迟或反转选择双人纵深、前景遮挡、高低差、负空间、反应持镜或有动机的固定镜头 |
| 慢节奏画面停滞 | 只写情绪或 locked-off，没有可见进展 | 补触发后的低幅呼吸、握力、视线、重心或道具动作，并让焦点、环境、光影或声音进入下一状态 |
| 道具瞬移或穿模 | 没有锁握点、分离状态和容器拓扑 | 写初始分离 -> 接近 -> 接触 -> 握紧/承重 -> 移动，并锁定开口、固定件、可动件和空气间隙 |
| 文字乱码 | 要求模型生成复杂文字 | 默认用无可读文字的画面结构；必要文字保持短句、单一位置和固定字形 |
| 血腥过度 | 接触点写成伤口细节 | 改为遮挡、反应、停机、静默、物体位置变化 |
| 光线空泛 | 只写暖光/冷光 | 补光源、方向、强度、阴影、材质反应 |
| 风格不稳定 | 风格名堆叠 | 编译为线条、色彩、光线、材质、构图 |

## 输出前最终检查

完整提示词至少满足：

- 有 `【基础风格】`。
- 每个独立段落都有 `【基础风格】`，不是“同上”。
- 有 `【角色/场景/素材锁定】` 或等价信息，且没有重复拆出 `【参考素材使用】`、`【场景锁定】`、`【角色锁定】`。
- 有 `【运镜与时间轴】`，且时间轴包含画面结构、焦点路径、动作和结尾状态。
- `【运镜与时间轴】` 使用已命名锁定，不频繁重复 @素材引用。
- 如果有台词、旁白、内心OS或画外音，它们都已经出现在 `【运镜与时间轴】` 的对应 CUT 中，而不是独立台词清单；默认不显示单句秒数。
- 剧本动作不足时，已经主动补充不改变剧情的目标驱动调度；没有用站桩覆盖，也没有为了动而随机加动作。
- 每句对白都有开口触发、说话中动作、语义反应和句尾新状态；多人镜头有分层、错峰、低幅次动作。
- 活动场景中的人物持续执行有目的任务；慢节奏或情绪/权力节拍使用正向低幅动作、画内变化和明确的下一状态。
- 有 `【声音设计】`，默认明确无背景音乐；用户要求 BGM 时按要求覆盖。
- 有 `【画面限制】`，固定保留“无字幕、无水印、无乱码文字；人物身份稳定，面部、身体和手部结构自然完整”。
- 逐句扫描演员行为：默认使用肯定句；把连续性改写成主体、部位/道具、路线和节奏。正向状态已锁定后删除同义否定句，固定技术清洁模板不计为演员缺席行为。
- 如果用户要求多个段落，每段都有独立提示词，而不是只写“同上”。
- 随机抽取任意一段单独阅读，首个 CUT 不需要标题、总说明或上一段也能明确人物、空间、道具、动作对象、路线和声音来源。
- 随机检查最终 CUT：只看到本段可见 OUT，不看到未来台词、动作、场景、角色或“为下一段”说明。
- 机位延续句使用正向状态元组；例如：`林夏的反应镜头严格复用CUT-001的摄影机位置、75mm焦段、右前方三分之四脸角度、人物大小和眼睛高度。`
- 对多机位场景逐镜执行“世界拓扑不变 + 摄影机投影重算”检查；随机抽取正拍、反拍和跨段首镜，门窗/座位/人物世界朝向一致，背景与 screen-left/right 符合当前摄影机几何。
- 如果用户只要最终提示词，不输出冗长分析。
- 如果是续写/下一段，确认已经使用实际接受结尾；如果缺少结尾证据，先请求用户提供。
- 如果使用静态末尾帧，确认锁定区写“只控制角色世界站位与相互距离”，场景来源另行声明，首镜明确 `hard cut/direct cut` 到新机位/新景别，且前中后景和 screen-left/right 已重算。
- 如果是失败修复，先判断 keep / post / edit / re-roll / rewrite，再给一个变量的修复版本。

## 输出修复句式

如果原提示词太抽象：

```text
把“镜头推进到危险处”改成 “slow dolly in from shoulder-height wide shot toward the fixed danger gap; focus racks from the driver's eyes to the metal pipe aligned with the handlebar.”
```

如果原提示词动作太乱：

```text
拆成 three beats: route -> contact point -> consequence. Do not ask the model to perform chase, fall, rescue and reaction in the same second.
```

如果原提示词切镜太密：

```text
Use 5-15s packed Seedance segments with internal markers; split only at strong scene, camera-readability, character/speaker, time-state, or completed-action boundaries.
```
