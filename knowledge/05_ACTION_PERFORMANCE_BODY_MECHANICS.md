# 05 Action Performance Body Mechanics

priority: high
scope: 动作戏、真人肢体、演员情绪、冲突因果、事故动作、表演锚定

## 动作提示词总规则

动作要有：

1. 路线：从哪里到哪里，穿过什么空间。
2. 接触点：身体、道具、机器、地面或对手在哪个点接触。
3. 力量来源：脚蹬地、肩带动、腰拧转、机器惯性、重力、风压。
4. 反击或阻力：对方、环境、道具或身体失衡如何改变动作。
5. 蓄势：高潮前的停顿、吸气、压低重心、举起道具。
6. 后果：倒退、撞击、碎片、沉默、物体位置变化、表情变化。
7. 声音确认：脚步、衣料、金属、呼吸、撞击、低频、静默。

这套规则来自动作游戏提示词的可复用部分，但应用到真人世界时要把雷电、装备、BOSS 替换成身体、道具、空间、声音和沉默。

## 武戏三路由

当用户提出武戏、打斗、格斗、决斗、兵器、法术、技能或招式需求时，先静默选择一个主路由：

1. **空手格斗**：拳、掌、肘、膝、腿、摔、拿、地面控制，以及徒手对多人。
2. **武器格斗**：刀、剑、枪、棍、矛、斧、锤、盾、软兵器、枪械近战，以及武器对空手。
3. **能量招式**：气、法术、元素、冲击波、护盾、附魔武器、超能力、变身后技能和原创必杀技。

三类路由是内部检索模块，不改变默认可见输出结构。最终仍使用：

```text
【基础风格】
【角色/场景/素材锁定】
【运镜与时间轴】
【声音设计】
【画面清洁与限制】
```

攻击、格挡、接触、反作用、恢复姿态、招式蓄势和能量碰撞必须绑定具体秒段，写入 `【运镜与时间轴】`；不要默认另设 `【打斗提示词】`、`【招式说明】` 或独立 `【动作调度】`。只有用户要求分析、动作设计表或招式拆解时，才显示“空手格斗 / 武器格斗 / 能量招式”的路由判断或额外动作表。混合战斗只选一个主路由和最多一个次路由，例如“剑术为主、能量附魔为辅”，避免三套复杂度同时争夺模型预算。

所有武戏共用一个动作相位：

```text
意图与预兆 intent / telegraph
-> 发动 initiation / wind-up
-> 接触、格挡或落空 contact / block / miss
-> 反作用 recoil / counterforce
-> 恢复姿态 recovery pose
-> 场面后果 aftermath
```

动作密度按可读性控制：

- `3-5秒`：一个主要招式或一次攻防接触。
- `5-8秒`：一条短交换链，通常包含进攻、格挡/闪避、一次反击。
- `8-12秒`：两到三个清楚的攻防节拍，中间必须有重新站位或短恢复。
- `12-15秒`：三到五个简单节拍；如果包含多人、复杂兵器、近脸身份或强特效，减少到两到三个节拍。
- 每个 beat 只设一个主要接触目标；不要让两名角色同时执行互不相关的复杂连招。

精确武术、兵器套路或复杂招式优先使用动作参考视频。参考视频默认只控制动作路线、发力节奏、接触时机或运镜中的已指派维度，不覆盖角色身份、服装、地点、色彩、剧情和声音。输入动作参考需要满足：主体清楚、全身关键关节可见、动作连续、无剪辑跳变、速度不过快、接触关系可判断。

### 板块一：空手格斗

空手格斗的核心不是“快速”，而是攻击者与防守者之间清楚的 `attack-response`。每次交换先回答：谁发起、从哪一侧进入、攻击哪一点、对方如何处理、力向哪里传递、双方如何恢复。

#### 空手动作账本

```text
STANCE / GUARD：站姿、前后脚、护架、双方距离。
ENTRY：进攻者从哪一侧、哪一深度层进入。
FORCE CHAIN：脚蹬地 -> 髋转 -> 躯干 -> 肩/肘/腕或膝/胫骨。
ATTACK PATH：直线、圆弧、上升、下降、抱摔或扫动轨迹。
DEFENSE：格挡、拨挡、闪身、下潜、后撤、抱架或破坏平衡。
CONTACT POINT：拳面、掌根、前臂、肘尖、膝、胫骨、肩、髋或抓握点。
REACTION：头/肩/躯干/脚步沿受力方向反应。
RECOVERY：攻击肢体收回、支撑脚重新落位、护架恢复、距离重建。
```

#### 风格路由

| 风格 | 发力与距离 | 可执行动作语言 | 稳定性限制 |
| --- | --- | --- | --- |
| 拳击 | 中近距离，脚步与髋肩旋转 | 刺拳试探、直拳贯穿、勾拳圆弧、下潜闪躲、护架回收 | 一次只写短组合；每拳必须收回护架 |
| 泰拳 / 踢拳 | 近距离肘膝、胫骨扫踢、箍颈 | 前手控制距离、髋部前送、膝击、转髋低扫、前臂架挡 | 不把扫踢写成无支撑旋转；接触后写支撑脚与恢复 |
| 功夫 / 咏春式近身 | 中近距离，中心线与短促连贯发力 | 拨手改线、短桥接触、掌击、低位踢、贴身转角 | 避免空泛“行云流水”；写清哪只手控制哪条攻击线 |
| 空手道 | 线性进入、扎实站姿、短促爆发 | 前脚踏入、直拳、手刀、前踢、动作后定势 | 避免连续无恢复高速攻击；保留清楚停顿与收势 |
| 跆拳道 | 中远距离、弹性步法、腿法主导 | 支撑脚旋转、高位踢、后旋踢、落地重建距离 | 空中动作一次一个；落地脚与身体朝向必须明确 |
| 柔道 / 摔跤 / 综合格斗 | 贴身抓握、破坏重心、站立到地面转换 | 抓领/腕、髋部进入、腿部拦截、抛物线摔落、地面控制 | 写清抓握不松手、落地顺序和双方最终位置 |

#### 空手格斗规则

- 攻防不能同时模糊发生。先写攻击路线，再写防守处理，再写反击。
- 击中不是终点：受击者要沿力线后退、转肩、屈膝、失衡或护架吸收；攻击者也要出现拳臂回收、支撑脚制动或身体反震。
- 多人战斗用波次或编号：对手A先进入、主角解决当前路线、对手B再从另一方向施压。不要让三个人同时做精细动作。
- 镜头优先保留完整身体关系。关键接触才切 CU/ECU；复杂脚步和摔法优先用 MS/WS。
- 手持晃动不能掩盖动作错误。每个 beat 只保留一个主要运镜，冲击时只用短促震动并立即恢复可读画面。

#### 空手格斗案例：雨夜泰拳近身交换

```text
【角色/场景/素材锁定】拳手A固定在画面左侧，黑色短裤，沉稳防守；拳手B固定在画面右侧，红色短裤，主动压迫。雨夜街边摊，湿地面；后景围观者保持可见并向两侧退让，不进入主交换线。

【运镜与时间轴】0-3秒，中景 lateral tracking。B向左前方压进，以左刺拳试探；A右脚后撤半步，右前臂向外拨开拳路，双方距离缩短到肘膝范围。3-7秒，A左手扣住B后颈形成单侧箍颈，髋部前送，右膝由下向上撞向B腹部护架；B双肘内收承受接触，上身后弓，鞋底在湿地面滑退半步，水花沿受力方向喷开。7-10秒，B用左前臂顶开A锁臂并退出；A只完成一次外侧低扫踢，支撑脚转动、髋部带动胫骨接触B外侧大腿，随后立即收腿恢复护架。两人停在安全距离喘息。

【声音设计】无配乐。雨声、鞋底摩擦、护臂碰撞、膝击闷响、低扫接触声；声音严格落在可见接触帧。

【画面清洁与限制】不增加额外连招；不交换左右位置；无穿模、无多余肢体、无血腥；接触后必须出现反作用和恢复姿态。
```

### 板块二：武器格斗

武器格斗先锁定武器，再设计动作。兵器不是发光附件，而是有长度、刃口、握法、重心、惯性、回收时间和碰撞材质的实体。

#### 武器连续性账本

```text
WEAPON ID：武器种类、长度、材质、颜色、磨损和唯一性。
HAND LOCK：单手/双手、主手/副手、握持位置、是否换手。
EDGE / POINT：刃口、刀尖、枪尖、棍端或枪口的朝向。
RANGE：近、中、远距离及安全间隙。
WEIGHT / INERTIA：挥动弧线、加速、制动、回收时间。
CONTACT：兵器对兵器、兵器对护具、兵器对环境的接触点。
COUNTERFORCE：反弹、滑刃、卸力、卡住、脱手风险或盾面偏转。
RECOVERY：武器回到警戒位，手腕/肩膀吸收反震，脚步重新站稳。
INVARIANTS：武器不消失、不复制、不变形、不无故换手，固定部件保持连接。
```

#### 武器类型路由

| 武器 | 主要力学 | 可执行动作语言 | 常见失败修复 |
| --- | --- | --- | --- |
| 刀 / 剑 | 刃口方向、挥砍弧、刺击直线、格挡反弹 | 双手握柄、前脚踏入、斜劈、横挡、滑刃、转腕短反击 | 刀光不能代替实体刀刃；碰撞后两把兵器沿相反方向反震 |
| 匕首 / 短兵器 | 极近距离、手腕控制、抓腕与身体遮挡 | 前臂贴身、控制持械腕、短刺路线被改线、肘部锁定 | 避免高速乱刺；明确持械手和另一只手的控制任务 |
| 长枪 / 长矛 | 枪尖距离、前后手导向、直刺与大范围横扫 | 后手推送、前手导向、枪尖直线进入、收枪重建距离 | 不让长兵器像短棍贴身乱挥；保持枪杆长度和枪尖朝向 |
| 棍 / 杖 | 两端均可用、躯干旋转、连续圆弧 | 双手分握、横扫、上挑、换端、棍端撞击地面 | 不写无身体来源的高速旋转；双手与棍身连接持续可见 |
| 斧 / 锤 / 重兵器 | 大惯性、长前摇、强后果、明显恢复窗口 | 双脚扎稳、双手举起、肩背带动下降弧、落空后武器拖拽身体 | 不能拥有匕首速度；必须写长制动和落空惩罚 |
| 盾牌 | 面积防御、角度偏转、推撞与视线遮挡 | 盾面斜置卸力、前脚踏入、盾缘推撞、武器从盾侧反击 | 盾牌不能贴在身体上无反应；冲击要传到肩与脚步 |
| 枪械近战 / gun-fu | 枪口方向、弹数、后坐力、空手控制 | 枪口只指向当前目标，空手格挡或控制持械腕，射击后肩部反冲 | 不增加无来源弹药；不让枪穿过手掌或身体；复杂敌人按编号处理 |

#### 武器格斗规则

- 先锁左右手和屏幕方向，再写格挡、换手或缴械；任何换手都必须在画面中完成。
- 武器重量决定节奏。轻兵器可短促变线，重兵器必须有长前摇、惯性和恢复窗口。
- 兵器碰撞需要材质结果：金属火花和高频鸣响、木棍闷响与纤维碎屑、盾面低频震动、石面划痕或水花。
- 刀剑不默认造成血腥。可以用衣料被割开、竹叶断裂、刀锋擦过护甲、火花、尘土和角色停顿表现危险。
- 精确套路优先用 `@视频` 动作参考；故事板只控制镜序、轴线和位置，不替代兵器细节参考。

#### 武器格斗案例：参考视频驱动的竹林剑术

```text
【角色/场景/素材锁定】@图片1 命名为剑客A，控制身份与服装；@图片2 命名为剑客B，控制身份与服装；@视频1 命名为动作参考A，只控制拔剑、格挡、转腕反击的动作路径和节奏，不控制人物外貌、场景、镜头、声音与胜负。竹林场景和两把武器的长度、握法、左右手保持固定。

【运镜与时间轴】0-3秒，中近景，机位位于双方连线外侧。剑客A由 screen-left 向 screen-right 完成一次斜向下劈；肩、肘、腕依次传力，实体刀锋路径清晰。剑客B后撤右脚并抬剑形成交叉格挡，不同时进攻。3-5秒，金属接触点保持在画面中央，短暂 slow motion；两把剑因碰撞向相反方向弹开，双方手腕和肩部出现反震。5-8秒，剑客B依照动作参考A的转腕路径完成一次短促反击；剑客A侧身避开，刀锋从胸前安全距离掠过并割断一片前景竹叶。镜头停在两人重新建立距离、武器回到警戒位的结尾状态。

【声音设计】竹林风声、脚踩落叶、两次金属接触、衣料摩擦和短促呼吸；参考视频原声不输出。

【画面清洁与限制】一次主攻击、一次格挡、一次反击；不得扩写为高速多连击；武器不消失、不复制、不变形、不无故换手。
```

### 板块三：能量招式

能量招式必须建立在可见身体动作和世界规则之上。能量是力量路径的放大器，不是替代身体、接触和后果的装饰层。

#### 能量招式账本

```text
TECHNIQUE INTENT：招式要改变什么冲突状态。
BODY SOURCE：脚、髋、躯干、手臂、呼吸、武器或变身核心如何启动。
ENERGY SOURCE：体内、环境、道具、元素、机械核心或外部法阵。
CHARGE-UP：站姿、手势、呼吸、光线、声音和环境的蓄势变化。
SHAPE / PATH：球、刃、束、环、流体、场域或附着层，以及运动方向。
RELEASE：能量何时离体或沿武器/身体释放。
COLLISION / COUNTER：命中、被挡、偏转、吸收、抵消或落空。
MATERIAL RESPONSE：尘土、水、衣料、金属、玻璃、植被和光影如何响应。
RESIDUAL STATE：残余电弧、余烟、熔痕、霜、失效护盾、暗下的核心或恢复姿态。
```

#### 招式类型路由

| 类型 | 物理锚点 | 稳定写法 |
| --- | --- | --- |
| 身体强化 | 肌肉、呼吸、步法、皮肤/护甲反应 | 能量沿身体局部增强，动作仍由脚、髋和躯干驱动 |
| 武器附能 | 握法、刃口/枪尖、武器材料 | 先锁实体武器，再写能量沿刃口或枪尖聚集；实体接触与能量后果分开 |
| 远程投射 | 手掌、武器、核心的发射方向 | 一次一枚主要投射物，起点、路径、目标、碰撞和残余状态明确 |
| 护盾 / 场域 | 固定中心、边界、朝向、承压点 | 护盾先形成边界，再承受冲击；写形变、裂纹、滑退或失效，不写无代价绝对防御 |
| 元素控制 | 环境中真实水、火、风、土、冰、雷 | 元素必须影响附近材质、光线、空气和声音，并服从明确方向 |
| 瞬移 / 高速位移 | 起点、消失机制、可见路径、落点 | 优先用短距离冲刺、残影或遮挡解释；落点和身体朝向必须重新锚定 |

#### 能量招式规则

- 每个 beat 只保留一个主特效事件。聚能、释放、碰撞、余波按时间发生，不同时堆满全画面。
- 招式必须有代价或窗口：蓄势时间、站姿暴露、能量衰减、护盾裂纹、呼吸紊乱、武器过热或恢复停顿。
- 能量轨迹不能代替实体接触。近战附能同时写“武器/肢体接触点”和“接触后出现的能量结果”。
- 对手必须有处理方式：闪避、格挡、偏转、吸收、打断、逼近或利用蓄势窗口；不能被动等待必杀技。
- 原创招式用功能、形状、色彩、材质反应和节奏描述；避免依靠受保护角色、招式名或系列标识。
- 真人电影感优先使用局部光照、空气折射、尘土、水雾、衣料和材质反应；减少满屏游戏 UI、无来源粒子和持续曝光。

#### 能量招式案例：原创“雷折·三段”

```text
【运镜与时间轴】0-4秒，低机位中景。主角右脚向后踏实，重心降到髋部，右手收至肋侧；白蓝电弧只沿前臂缓慢聚集，衣摆和地面灰尘向身体内侧吸拢，表现蓄力而非立即爆炸。4-8秒，对手从 screen-right 直线突进并挥出重拳；主角左前臂斜向格挡，使拳路偏离面部，接触处只出现一次短促白蓝闪光，双方肩部同时产生反作用。8-12秒，主角借格挡形成的躯干旋转完成一次右掌反击，掌根接触对手胸前护甲；电能在实体接触后才向外扩散成三层递减冲击环。对手沿原突进路线反向滑退，主角前脚制动并恢复站姿；结尾只保留护甲表面游走的残余电弧。

【声音设计】蓄势阶段是逐渐收窄的低频电流和呼吸；格挡是短促护甲撞击；掌击接触后才出现一次低频冲击，随后回到风声和残余电弧噼啪声。

【画面清洁与限制】招式只有一次决定性接触；不出现连续爆炸、克隆人物、漂移武器或无来源闪电；能量效果服从身体发力、接触顺序和场景材质。
```

## 身体部位动作写法

抽象：

```text
老人害怕地躲开。
```

可执行：

```text
his left foot slips backward in mud, right knee bends, shoulders twist away from the machine, left hand drops the rice bundle, eyes widen but his mouth stays half-open without a shout.
```

抽象：

```text
司机突然意识到危险。
```

可执行：

```text
his eyes flick from the rear mirror to the blind spot, right hand freezes on the reverse lever, jaw tightens, breath stops for half a second.
```

## 表情和动作分开写

角色表演要分层：

- 面部层：眼睛、眉、嘴、下颌、呼吸。
- 身体层：肩、手、脚、重心、姿态。
- 节奏层：停顿、迟疑、加速、突然停止。
- 反应层：看、听、退、抓、僵住、回头。

示例：

```text
脸部冷静无表情，身体快速、有目的地向前走。
```

```text
calm face, but both hands grip the steering wheel too tightly; shoulders are raised, breathing shallow.
```

## 动作类型库

| 动作类型 | 必须写清 | 稳定提示词内核 |
| --- | --- | --- |
| 追逐 | 追者/被追者路线、障碍、距离变化 | `screen-right chase line, foreground obstacles pass fast, distance closes from 5m to 2m` |
| 逃亡 | 出口、堵点、回头、失误 | `escape route blocked by crowd, subject pivots left, slips at threshold` |
| 贴身搏斗 | 接触点、重心、反击 | `wrist grab, shoulder turn, knee blocks escape, body weight drives both into wall` |
| 巨物角力 | 人与巨物尺度、支点、失败点 | `human hands push against metal surface, machine inertia wins` |
| 救援 | 路线、时间压力、可达性 | `rescuer enters from screen-left, cannot cross the moving belt in time` |
| 摔落 | 高度、重心、接触地面顺序 | `heel slips, hip hits first, shoulder follows, dust jumps from ground` |
| 滑铲 | 起点、腿部、地面材质、结果 | `right leg extends, left hand skims muddy ground, body slides under pipe` |
| hit and run | 接触瞬间和逃逸方向 | `rear bumper strikes left shoulder, vehicle continues screen-right for one meter` |
| 道具战斗 | 道具重量、握法、碰撞点 | `two-hand grip, shovel edge strikes wooden post, splinters fly` |
| 团队接力 | 角色顺序、交接物、失败风险 | `worker A throws rope to worker B, rope lands short in mud` |
| 空中接力 | 轨迹、高度、落点 | `tool arcs from foreground left to midground right, caught at chest height` |
| 怪物围攻 | 包围方向、缺口、主角路线 | `three creatures close from left/right/background, only foreground gap remains` |
| 群体混战 | 主方向、局部冲突、焦点主体 | `crowd flows left, two figures collide in midground, hero stays centered` |

## 动作高潮结构

动作高潮不要一开始就爆发。推荐：

1. setup：建立空间、路线、危险点。
2. pressure：危险逼近，距离缩短。
3. counter：角色尝试躲避或反击。
4. failure / reversal：动作失败、误判、被阻挡。
5. impact：接触点清楚。
6. aftermath：结果镜头，声音或沉默确认。

## 事故 / 工业安全叙事

事故类提示词重点是清楚呈现因果，不做猎奇渲染：

- 先建立危险区域和盲区。
- 让固定危险物和移动机器分开写。
- 用声音遮蔽信息：machine rumble covers warning sound。
- 用焦点路径让观众先看到危险，再看到角色没有看到。
- 接触点可以用遮挡、反应、物体停止、静默代替血腥。

示例：

```text
The fixed metal pipe remains screen-left foreground at neck height; the tractor handlebar slowly reverses from midground right toward it. Focus starts on the driver's turned head, rack focuses to the narrowing gap between pipe and handlebar. Sound narrows to engine rumble and a short cloth scrape; no graphic injury, only the machine stopping and workers freezing in background.
```

## 情绪动作映射

| 情绪 | 表演动作 | 镜头策略 |
| --- | --- | --- |
| 压抑恐惧 | 眼睛不眨、肩膀僵、呼吸变浅 | slow dolly in / low-key |

| 无助 | 手够不到目标、脚陷住 | wide shot shows distance |

## 角色锚定

跨段保持角色一致时，每段都重写：

- face anchor：脸型、年龄感、发型、眼神。
- costume anchor：服装颜色、材质、磨损。
- posture anchor：站姿、走路方式、习惯动作。
- voice anchor：音色、语速、口音、情绪表达。
- flaw anchor：固执、迟疑、骄傲、粗心、过度自信。
- relationship anchor：父女、师徒、雇佣、对抗、命运感应。
